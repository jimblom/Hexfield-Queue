# Hexfield Queue Schema — v0.1

> Formal specification for `.queue.yaml` files.

---

## Overview

A `.queue.yaml` file describes a playlist in a service-agnostic, human-readable format. It is designed to be written by hand, generated by AI, or imported from a music service. Export adapters translate it to service-specific formats.

**File extension:** `.queue.yaml`

---

## Top-Level Structure

```yaml
version: "0.1"
meta:
  # playlist metadata
tracks:
  # ordered list of track or album entries
```

---

## `version`

| | |
|---|---|
| Required | yes |
| Type | string |
| Current value | `"0.1"` |

Schema version. Quoted string, not a number. Bump for breaking changes.

---

## `meta`

Playlist-level metadata.

### `meta.title`

| | |
|---|---|
| Required | yes |
| Type | string |

Human-readable playlist name. Used as the output filename when exporting.

### `meta.description`

| | |
|---|---|
| Required | no |
| Type | string |

Intent or context for the playlist. Shown in the Hexfield Deck playlist header.

### `meta.created`

| | |
|---|---|
| Required | no |
| Type | ISO 8601 date string (`YYYY-MM-DD`) |

Date the queue file was created.

### `meta.tags`

| | |
|---|---|
| Required | no |
| Type | string[] |

Genre, mood, occasion, or other categorization tags.

```yaml
tags: ["rock", "blues-rock", "introduction"]
```

### `meta.generated_by`

| | |
|---|---|
| Required | no |
| Type | enum |
| Values | `"human"` \| `"claude"` \| `"import"` |

Indicates the origin of the queue file. Makes every file self-documenting.

### `meta.prompt`

| | |
|---|---|
| Required | no |
| Type | string (multiline) |

The original natural language prompt used to generate this queue, if `generated_by: "claude"`. Preserved for reproducibility and context.

```yaml
prompt: |
  Introduce me to Black Pistol Fire. Start with their latest release...
```

## `tracks`

An ordered array of track or album entries. The order defines playback sequence.

### Common Fields (all track types)

| Field | Required | Type | Notes |
|---|---|---|---|
| `type` | yes | enum | `"track"` or `"album"` |
| `title` | yes | string | Track or album title |
| `artist` | yes | string | Primary artist name |
| `year` | no | integer | Release year |
| `notes` | no | string | Curatorial annotation — displayed in Deck card |

### `type: track`

A single song.

| Field | Required | Type | Notes |
|---|---|---|---|
| `album` | yes* | string | Album the track appears on. Required for `type: track`. |

```yaml
- type: track
  title: "Suffocation Blues"
  artist: "Black Pistol Fire"
  album: "Big Beat '59"
  year: 2014
  notes: "Their most popular track by streams."
```

### `type: album`

A full album, intended to be played in sequence. No `album` field — the `title` is the album title.

```yaml
- type: album
  title: "Deadwater"
  artist: "Black Pistol Fire"
  year: 2023
  notes: "Most recent full album — play in full."
```

---

## Validation Rules

1. `version` must be present
2. `meta.title` must be present and non-empty
3. `tracks` must be a non-empty array
4. Each track must have `type`, `title`, and `artist`
5. `type` must be `"track"` or `"album"`
6. `tracks` with `type: track` must include an `album` field

Run `node scripts/validate.js <file>` to check a file against these rules.

## Complete Example

```yaml
version: "0.1"
meta:
  title: "Introduction to Black Pistol Fire"
  description: "Start with latest, then their best album, then most popular, then influences."
  created: "2026-03-26"
  tags: ["rock", "blues-rock", "introduction"]
  generated_by: "claude"
  prompt: |
    Introduce me to Black Pistol Fire.

tracks:
  - type: track
    title: "Railhead"
    artist: "Black Pistol Fire"
    album: "Deadwater"
    year: 2023
    notes: "Latest single as of generation date."

  - type: album
    title: "Deadwater"
    artist: "Black Pistol Fire"
    year: 2023
    notes: "Most recent full album — play in full."

  - type: track
    title: "Suffocation Blues"
    artist: "Black Pistol Fire"
    album: "Big Beat '59"
    year: 2014
    notes: "Their most popular track by streams."
```
