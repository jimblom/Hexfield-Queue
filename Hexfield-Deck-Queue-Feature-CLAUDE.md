# Hexfield Deck — Queue Card Feature Brief
## For inclusion in / appended to Hexfield-Deck's CLAUDE.md

> This section documents the Hexfield Queue integration — a new card type added to Hexfield Deck
> that renders `.queue.yaml` playlist files as interactive, draggable song/album cards.

---

## Feature: Queue Card Rendering

### What Is a Queue Card?

A Queue Card is a new Hexfield Deck card type, distinct from the existing planner/task cards. It is rendered when Deck opens or detects a `.queue.yaml` file (see Hexfield Queue repo for full schema spec).

Deck's core principle — **the file is the source of truth, the visual is a layer** — applies here exactly as it does for markdown planner files. Reordering cards in the webview updates the `tracks:` array order in the source YAML.

---

## Triggering the Queue View

Deck should activate queue rendering when:
- A file with `.queue.yaml` extension is opened in VS Code
- OR a file begins with `version:` and contains a `tracks:` key (fallback detection)

The queue view opens in a webview panel, same as the existing Deck board view.

## Card Design Spec

### Track Card (`type: track`)

```
┌─────────────────────────────────────┐
│ ≡  [track]                          │  ← drag handle + type badge
│                                     │
│  Suffocation Blues                  │  ← title (large)
│  Black Pistol Fire                  │  ← artist
│  Big Beat '59 · 2014                │  ← album · year
│                                     │
│  Their most popular track by        │  ← notes (small, italic, optional)
│  streams.                           │
└─────────────────────────────────────┘
```

### Album Card (`type: album`)

```
┌─────────────────────────────────────┐
│ ≡  [album]                          │  ← drag handle + type badge (different color)
│                                     │
│  Deadwater                          │  ← album title (large)
│  Black Pistol Fire · 2023           │  ← artist · year
│                                     │
│  Most recent full album —           │  ← notes
│  play in full.                      │
└─────────────────────────────────────┘
```

### Visual Conventions
- Type badge: `track` = neutral, `album` = accent color (distinguishes full-album entries)
- `notes` field: rendered below in smaller italic text, only shown if present
- Drag handle (`≡`) on left edge — same UX as existing Deck cards
- Sequential numbering (1, 2, 3...) displayed on left or top of card

## Playlist Header

Above the cards, render the queue `meta` as a playlist header:

```
┌─────────────────────────────────────────────┐
│  Introduction to Black Pistol Fire          │  ← meta.title
│  Start with latest, then best album...      │  ← meta.description (truncated)
│  5 tracks · Generated by Claude             │  ← count + meta.generated_by
│                                    [Export] │  ← export button
└─────────────────────────────────────────────┘
```

---

## Bidirectional Sync

When the user drags to reorder cards, the `tracks:` array in the source `.queue.yaml` is rewritten to match the new order. This is the same bidirectional sync pattern used for Deck's markdown kanban columns.

Implementation note: preserve all YAML fields, comments, and `meta` block — only reorder the `tracks:` sequence items.

---

## Export Button

The `[Export]` button in the playlist header triggers the Hexfield Queue export pipeline:
- MVP: generates and downloads a CSV for Apple Music import
- The export logic lives in the `Hexfield-Queue` repo as a script
- Deck invokes it either via a bundled copy or a Node.js child process

---

## File Association

Register `.queue.yaml` as a file type Deck handles in `package.json`:

```json
"contributes": {
  "languages": [{
    "id": "hexfield-queue",
    "extensions": [".queue.yaml"],
    "aliases": ["Hexfield Queue", "queue.yaml"]
  }]
}
```
